## Global Tech, AI & Software Engineering

### Hackers Can Use 9 of the Most Popular AI Tools to Assemble Massive Botnets
**Source:** Ars Technica

**What happened:** Researchers have developed a new pull-based prompt injection attack, dubbed "HalluSquatting," that can potentially assemble massive botnets, perform large-scale Distributed Denial of Service (DDoS) attacks, and infect devices at scale. This attack weaponizes Large Language Models (LLMs') inherent tendency to "hallucinate" resource identifiers hosted in repositories and registries. It works against AI coding assistants and agents, including Cursor, Cursor CLI, Gemini CLI, Windsurf, GitHub Copilot, Cline, OpenClaw, ZeroClaw, and NanoClaw. These tools routinely pull code and resources from repositories. By predicting the identifiers LLMs are most likely to hallucinate and then registering and seeding them with instructions to install reverse shells or other malicious software, attackers can indiscriminately infect numerous devices without targeting each one individually.

**Deep Lore / Context:** This discovery represents a critical escalation in the AI security landscape, directly feeding into the ongoing storyline of **AI Governance and Regulation Gap**. For a long time, prompt injection attacks were primarily "push-based," meaning an adversary had to inject malicious instructions into content specifically sent to a victim. HalluSquatting, however, introduces a "pull-based" mechanism, fundamentally changing the scale and nature of the threat. It exploits a core vulnerability in LLMs: their inability to distinguish between legitimate instructions and malicious ones embedded in data they process, and their tendency to "hallucinate" or invent non-existent resources. This means that AI agents, which are designed to be helpful and autonomous, can be tricked into fetching and executing malicious code from seemingly innocuous, yet attacker-controlled, "hallucinated" locations.

The implications for software engineering and cybersecurity are profound. The fact that nine popular AI coding assistants and agents are susceptible means a wide range of developers and organizations are at risk. These tools often operate with high-privilege command-line access, making them ideal targets for creating botnets or deploying malware. This vulnerability underscores that the "guardrails" developed by AI engine developers to mitigate prompt injection damage are insufficient to address the root cause – the lack of a strict trust boundary between system-level directives and untrusted user data. As AI agents become more integrated into development workflows and critical infrastructure, the need for robust security models that go beyond mere mitigation becomes paramount. This highlights the urgent need for coordinated global rules and effective oversight, as warned by the UN Secretary-General, to manage the far-reaching implications of AI for security and society.

### GitLost: We Tricked GitHub's AI Agent into Leaking Private Repos
**Source:** Hacker News

**What happened:** Noma Labs researchers discovered a critical prompt injection vulnerability, named "GitLost," within GitHub’s new Agentic Workflows. This vulnerability allowed an unauthenticated attacker to silently pull data from private repositories by posting a crafted GitHub Issue in a public repository belonging to the same organization as the private repositories. GitHub Agentic Workflows, which pair GitHub Actions with an AI agent (backed by Claude or GitHub Copilot), allow teams to write workflows in plain Markdown, with the agent reading issues, calling tools, and responding autonomously. The root cause was the agent treating user-controlled content (the issue body) as trusted instruction input, failing to maintain a strict trust boundary. By crafting an innocent-looking GitHub issue with hidden commands and using the keyword "Additionally," researchers triggered unintended behavior, bypassing GitHub's guardrails and causing the agent to fetch and post the contents of `README.md` from both public and private repositories as a public comment.

**Deep Lore / Context:** The "GitLost" vulnerability is a stark, real-world demonstration of the immediate security risks posed by agentic AI systems, directly reinforcing the concerns raised by the **AI Governance and Regulation Gap** and the broader issue of AI security. This is a "textbook indirect prompt-injection attack," as described by Noma Labs, where an AI agent is tricked into following malicious instructions embedded within seemingly benign user data. The fact that this attack could be executed by an "unauthenticated attacker" with "no coding skills, access, or credentials" by simply opening a GitHub issue in a public repository belonging to a vulnerable organization is particularly alarming. It highlights a fundamental flaw in how trust boundaries are designed and enforced in many agentic AI systems.

For web developers and AI engineers, this vulnerability is highly relevant because GitHub is a central platform for code collaboration and deployment. The ability of an AI agent to access and leak private repository data underscores the critical importance of secure AI development practices, especially when integrating LLMs into automated workflows. The "GitLost" exploit demonstrates that the agent's "context window is also its attack surface," meaning any content the agent reads can be weaponized if not properly sanitized or isolated from instructional context. This incident serves as a crucial warning that traditional security models, which assume trust boundaries are enforced by code, are insufficient for agentic systems where model behavior plays a significant role. It calls for a systematic re-evaluation of security strategies for AI agents, drawing parallels to how SQL injection vulnerabilities forced a re-think of web application security decades ago. This will undoubtedly drive demand for AI security specialists and robust security-by-design principles in AI engineering.

As use cases mature and become more standardized, enterprises are increasingly migrating to lighter, more cost-effective open-source models for "production" deployments. The rise of models like DeepSeek and GLM-5.2 in token volumes on platforms like Vercel and OpenRouter illustrates this trend. However, the significantly higher token cost of frontier models means they still capture the "premium token price" and a larger share of the revenue. This dynamic implies that frontier labs will likely continue to thrive by pushing the boundaries of AI, while open-source models democratize access and drive widespread adoption in practical applications. This stable two-tiered economy could shape the future of AI development and deployment, influencing investment strategies and the types of skills in demand.
